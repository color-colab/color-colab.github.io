How it's gonna be done

The AR experience will take place through a the join use of an iPad and a “magnifying glass”. 
Both of these items will be available in a stand next to the sculpture. In the ipad the initial video will show and then the color palette will appear.
The stand is going to be placed 0.5 meters away from the statue — close enough to signal it is related to this artifact specifically, but also far enough to let users stand between them to explore the statue itself. The stand will not be screwed into the ground, as it will be made of steel and thus heavy enough to not be knocked over. The iPad and the magnifying glass will not be tied to the stand. Rather, they will have a chip system that will ring if the user takes them more than 3 meters away from the stand (and the user will be warned about this in the stand itself —“Don’t take me too far away from my home”). The iPad will not have a lock, so the user can immediately access it.

Digital Data

The data to create the ColorColab in itself will be a series of image-based 3d models of sculptures, with an additional layer of texture that is customizable.
????????????????

The main digital data for user interaction will come from the ColorColab website server. A relational database framework will be employed to store the HEX color code, the RGB color code and an image of the color for the whole color wheel. Each of the hex codes can have additional textual notes, as for example if it is representative of an ancient pigment as supported by scholarly research. Additionally, the database will be enriched with user-inputted pictures and color tones, which will also have additional textual notes. This database of colors will the basis of another database surrounding color palettes. This database will simply be a list of pointers to the color database. Each palette will also have textual a pointer to a textual database with descriptions of the palettes if they are pre-defined, and with user notes if they are user-uploaded.
